## Project Goal

モダンで先進的な UI/UX のぷよぷよ (Puyo Puyo) ゲームアプリを Next.js で構築する。

## Commands

```bash
npm run dev      # 開発サーバー起動 (http://localhost:3000)
npm run build    # プロダクションビルド
npm run start    # プロダクションサーバー起動
npm run lint     # ESLint 実行
```

## Stack

- **Next.js 16** — App Router (`src/app/`)
- **React 19**
- **TypeScript** (strict mode)
- **Tailwind CSS v4** — PostCSS プラグイン経由 (`@tailwindcss/postcss`)
- パスエイリアス: `@/*` → `src/*`

## Architecture

### ディレクトリ構成方針

ゲームロジックと UI を分離して管理する:

```
src/
  app/              # Next.js App Router (ページ・レイアウト)
  components/       # UIコンポーネント (ゲームボード、ぷよ、スコア等)
  hooks/            # カスタムフック (ゲームループ、入力処理等)
  lib/              # ゲームロジック (純粋関数。Reactに依存しない)
    game/           # ぷよぷよのコアロジック (落下、連鎖判定、消去等)
  types/            # TypeScript 型定義
```

### ゲームロジック設計方針

- `src/lib/game/` 内のロジックは **純粋関数** で実装し、React に依存しない
- ゲーム状態は `useReducer` で管理し、アクションで状態遷移を表現する
- アニメーション・エフェクトは CSS アニメーションと Tailwind を活用する
- ゲームループは `requestAnimationFrame` または `setInterval` をカスタムフックでラップして使用する

### UI/UX 方針

- グラスモーフィズム・ネオモーフィズム等のモダンなデザイン言語を採用する
- ぷよの色は鮮やかなグラデーションで表現し、視覚的インパクトを重視する
- スコア加算・連鎖発生時にはパーティクルエフェクトやアニメーションを実装する
- レスポンシブ対応 (モバイルでのタッチ操作も考慮)

## 要件定義

### 1. ゲーム概要

ぷよぷよのルールに基づいたシングルプレイヤー向けパズルゲーム。  
フィールドにぷよを積み上げ、同色4つ以上を繋げて消し、連鎖を重ねてハイスコアを狙う。

---

### 2. フィールド仕様

| 項目 | 値 |
|------|-----|
| フィールドサイズ | 6列 × 12行 (標準ぷよぷよ準拠) |
| 非表示行 (天井) | 上部に2行の隠し行（ぷよが積み上がり判定に使用） |
| セル | 各セルはぷよ1個分 |

---

### 3. ぷよ仕様

#### 3.1 色の種類
- 赤・青・緑・黄・紫 の5色
- 各色はグラデーション表現で視覚的に区別する

#### 3.2 ツモ (操作ぷよ)
- 2個1組のペアでフィールド上部中央から出現
- 「軸ぷよ」と「副ぷよ」で構成され、軸ぷよを中心に副ぷよが回転する

#### 3.3 NEXT表示
- 次のツモを2セット先まで表示する

---

### 4. 操作仕様

#### 4.1 キーボード操作

| キー | 動作 |
|------|------|
| `←` / `A` | 左移動 |
| `→` / `D` | 右移動 |
| `↓` / `S` | 高速落下 (ソフトドロップ) |
| `Space` / `↑` | 一瞬で最下段まで落下 (ハードドロップ) |
| `Z` / `,` | 左回転 (反時計回り) |
| `X` / `.` | 右回転 (時計回り) |

#### 4.2 モバイル操作
- スワイプ左右: 移動
- スワイプ下: ソフトドロップ
- タップ左半分: 左回転、タップ右半分: 右回転

---

### 5. ゲームロジック仕様

#### 5.1 落下
- 一定間隔 (レベルに応じて加速) でツモが1行ずつ落下する
- 落下速度の初期値: 1000ms/行、レベルアップごとに短縮

#### 5.2 設置
- ツモがフィールド底面または他のぷよの上面に接触した時点で設置確定
- 設置後、空中に浮いたぷよは重力落下する (連鎖落下)

#### 5.3 消去判定
- 同色のぷよが上下左右に4個以上連結した場合、消去対象となる
- 消去は設置 → 浮きぷよ落下 → 消去判定 のサイクルを連鎖が発生しなくなるまで繰り返す

#### 5.4 連鎖
- 消去 → 落下 → 再消去 の繰り返しを「連鎖」として計上する
- 連鎖数に応じて倍率が上昇する

#### 5.5 おじゃまぷよ (将来拡張用)
- 現バージョンでは実装しないが、型定義に `ojama` カラーを含めておく

---

### 6. スコア計算

```
消去スコア = 消去ぷよ数 × 10 × 連鎖ボーナス × 色ボーナス × 連結ボーナス
```

| 連鎖数 | 連鎖ボーナス |
|--------|------------|
| 1 | 0 (ボーナスなし) |
| 2 | 8 |
| 3 | 16 |
| 4 | 32 |
| 5以上 | 前の値 × 2 |

- **色ボーナス**: 同時消去色数に応じて加算 (1色: 0, 2色: 3, 3色: 6, 4色: 12, 5色: 24)
- **連結ボーナス**: 同色の連結数が4超の場合、超過1個につき +1

---

### 7. レベルアップ

- 一定個数のぷよを消すごとにレベルアップ (初期値: 30個/レベル)
- レベルアップで落下速度が上昇し、難易度が増す
- 最大レベル: 15

---

### 8. ゲーム終了条件

- ツモ出現位置 (フィールド中央上部) がすでにぷよで塞がれた場合にゲームオーバー

---

### 9. UI/UX 要件

#### 9.1 画面構成
- **メイン画面**: ゲームフィールド (中央) + スコアパネル (右) + NEXT表示 (右上)
- **スタート画面**: タイトルロゴ + スタートボタン + ハイスコア表示
- **ゲームオーバー画面**: スコア表示 + リトライボタン + ハイスコア更新演出

#### 9.2 ビジュアル
- 背景: 深い宇宙・グラデーション等のダークテーマを基調
- フィールド: グラスモーフィズム (半透明パネル + ブラー)
- ぷよ: 色ごとにグラデーション + ドロップシャドウ + 丸みのある形状 (border-radius)
- 連鎖発生時: 画面フラッシュ + 連鎖数テキストのポップアップアニメーション
- 消去時: ぷよが弾けるパーティクルエフェクト

#### 9.3 アニメーション
- ぷよ落下: スムーズなトランジション
- 設置: 軽い衝撃エフェクト (スケール変形)
- 消去: フェードアウト + 爆発エフェクト
- 連鎖テキスト: スケールアップ → フェードアウト

#### 9.4 サウンド (将来拡張用)
- 現バージョンでは実装しないが、サウンドマネージャのインターフェースを定義しておく

---

### 10. データ永続化

- ハイスコアを `localStorage` に保存する
- 保存キー: `puyo-highscore`

---

### 11. 非機能要件

- **フレームレート**: 60fps を維持する (重い処理は `requestAnimationFrame` で管理)
- **アクセシビリティ**: キーボード操作のみでゲームが完結できること
- **ブラウザ対応**: Chrome / Firefox / Safari / Edge 最新版
