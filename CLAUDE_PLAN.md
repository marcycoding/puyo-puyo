# ぷよぷよゲームUI／機能の改善・追加プラン

これまでの実装で、ぷよの落下・回転・消去・連鎖・スコア計算といった主要なゲームロジックはほぼ完成しています。
本ステップでは、より「ぷよぷよらしい」プレイ感とモダンなUI/UXを実現するために、以下の改善を行うことを提案します。

## 1. ぷよの連結表現（UIの向上）

オリジナルのぷよぷよの最大の特徴として「同じ色が上下左右に並ぶと、互いにくっついて形が変わる（スライムのように見える）」という視覚的表現があります。
現状はすべて独立した丸い円（`rounded-full`）で描画されていますが、これを周囲の同色ぷよの有無から計算し、CSSの `border-radius` を動的に変更することで、シームレスに繋がっているようなリッチなUI表現を実装します。

## 2. 全消し（All Clear）ボーナスの実装

連鎖が終了した時点で盤面にぷよが1つも残っていない場合、「全消し（All Clear）」を達成したと判定し、アニメーション演出と共にボーナススコアを加算する機能を実装します。

## 3. サウンド・Audioマネージャー基盤の追加

`CLAUDE.md` に記載がある「将来拡張用」のサウンドマネージャーインターフェースを実装します。
すぐに音素材を用意しなくても、例えばぷよが「消える瞬間」「連鎖した瞬間」に `playSound('chain')` のように呼び出せる関数・フックを用意しておき、拡張性を持たせます。あるいはWeb Audio APIを用いた簡単な電子音（ピコッ、パポン等）でプレースホルダー音を鳴らすことも可能です。

## 4. その他 描画微調整

- ハードドロップ時や接地（Lock）時などのアニメーションの調整
- モバイル操作等のさらなる最適フィードバック（必要であれば）
